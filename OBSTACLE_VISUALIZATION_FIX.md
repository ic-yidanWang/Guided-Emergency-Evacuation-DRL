## 障碍物可视化问题修复总结

### 问题 1：障碍物未正确可视化
**根本原因**：
- 在 `visualization.py` 和 `run_guided_visualize.py` 中，障碍物的坐标处理不一致
- 配置文件中的障碍物坐标是**绝对坐标**，但代码错误地乘以了 domain 倍数
- 这导致障碍物显示在错误的位置或不显示

**修复**：
- 删除了不必要的 domain 缩放乘法
- 确保障碍物坐标直接使用绝对坐标（已从配置文件中读取）

修改文件：
- `evacuation_rl/utils/visualization.py` - line 174-196, 339-361
- `run_guided_visualize.py` - line 137-167

### 问题 2：障碍物影响范围远大于自身大小
**根本原因**：
- `Ob_size` 被设置为障碍物的**可视化半径**
- 碰撞检测使用 `Ob_size` 作为碰撞检测半径
- 导致碰撞范围与可视化大小不匹配，agents 在看不到的范围就开始躲避

**修复**：
- 圆形障碍物的碰撞半径设为视觉大小的 80%：`size * 0.8`
- 这样障碍物在视觉上和碰撞检测上的大小更接近
- 对矩形障碍物保持现有的小碰撞半径 0.03

修改文件：
- `run_guided_visualize.py` - line 232-234

### 坐标系统说明
- **配置文件（JSON）**：所有障碍物坐标和大小都是**绝对坐标**（例如 0-10）
- **保存文件（CFG）**：保存时转换为**归一化坐标** [0,1]
- **可视化**：使用配置文件中的绝对坐标直接绘制

### 测试结果
✓ 障碍物现在正确显示在配置文件指定的位置
✓ 障碍物的碰撞范围与可视化大小更匹配
✓ Agents 的行为与障碍物的显示位置一致
